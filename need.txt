## 项目名称：角色扮演聊天机器人（情感陪伴 / 游戏 NPC 方向）

### 一、项目背景与目标

**背景**  
当前大模型对话产品多为通用助手，缺乏鲜明、稳定、可长期相处的“角色”。用户需要能与具有特定人设、语气、记忆和世界观的 AI 角色进行沉浸式交流，用于情感陪伴、游戏 NPC 交互、IP 角色再现等场景。

**核心目标**  
构建一个**可配置角色、可长期记忆、高度拟人**的聊天机器人，支持 Web、小程序或 Discord 等多端接入。角色需具备一致性（不会“出戏”）、渐进式记忆（记得用户说过的话），并能根据角色背景知识进行合理回答。

### 二、用户角色与使用场景

| 用户类型 | 使用场景举例 | 关键诉求 |
|---------|-------------|----------|
| 普通用户 | 心情低落时找“温柔学姐”聊天；睡前与“童话精灵”互动 | 情感安慰、被倾听、角色真实感 |
| 游戏玩家 | 在开放世界 RPG 中与酒馆老板、同伴 NPC 对话，触发任务或彩蛋 | 角色符合世界观、对话有游戏关联信息 |
| 创作者 | 为自己小说中的角色构建 AI 版本，供读者体验 | 角色设定自由导入、语气风格可调 |
| 教育者 | 扮演历史人物（孔子、爱因斯坦），向学生讲解知识 | 知识准确、不违背史实、有互动问答 |

### 三、功能性需求

#### 1. 角色定义与管理
- **角色创建**：支持通过“角色卡”方式创建新角色，字段包括：  
  - 姓名、年龄、性别  
  - 性格标签（如温柔、毒舌、乐天派）  
  - 背景故事（自由文本）  
  - 说话风格（喜欢用比喻/短句/方言/标点习惯）  
  - 禁忌话题（拒绝谈论的内容或安全红线）
- **角色编辑**：管理员或创作者可随时修改角色设定，修改后聊天一致性自动更新。
- **角色广场**：用户可以公开自己创建的角色，并试玩他人角色。

#### 2. 对话与记忆
- **短时记忆**：同一会话中，记住最近至少 20 轮对话内容，不偏离上下文。
- **长时记忆**：跨会话记住用户主动提及的重要信息（例如：“我叫小夏”“我怕黑”“我养了一只猫叫球球”）。  
  - 系统应能自动判断信息重要性，也可由用户明确说“记住这个”。
  - 支持用户手动管理记忆（查看、删除某条记忆）。
- **记忆召回**：当新话题与历史记忆相关时，角色应主动引用记忆（如“你之前说怕黑，现在还好吗？”）。

#### 3. 角色行为约束
- **角色一致性**：不能出现违反角色人设的言行（例如温柔的护士不能说脏话）。提供“一致性评分”后台监控。
- **安全合规**：禁止生成色情、暴力、违法内容；必须遵守国家法规与平台内容准则。
- **知识边界**：角色只知道其设定背景内的知识（例如古罗马士兵不会知道智能手机）。超出边界时，应给出符合人设的回避回答（如“我只是个铁匠，不太懂这些……”）。

#### 4. 多模态与交互扩展（可选但建议）
- **情感表达**：角色支持发送表情符号、图片（基于角色风格的图片库）或简短语音。
- **动作描写**：在文字回复中加入「*」动作，如「（轻轻推来一杯热茶）」。用户可设定是否显示动作。
- **环境感知**（游戏场景）：若接入游戏引擎，角色能感知当前游戏时间、地点、天气等，并改变对话。

### 四、非功能性需求

| 指标 | 要求 |
|------|------|
| 响应时间 | 首字延迟 ≤ 2 秒（不含语音合成），完整回复 ≤ 5 秒 |
| 并发能力 | 支持单角色同时与 1000 名用户对话无明显排队 |
| 角色一致性 | 随机抽测 100 条对话，违反人设比例 < 5% |
| 记忆准确率 | 用户明确要求记住的信息，下次召回成功率 > 90% |
| 多轮对话长度 | 至少支持 32k tokens 上下文（约 2 万字对话历史） |
| 可用性 | Web/小程序界面需适配移动端，对话历史云端保存至少 30 天 |
| 数据隐私 | 用户与角色的对话内容加密存储，允许用户导出或删除全部数据 |

### 五、约束与限制

- **模型选择**：可使用闭源 API（OpenAI、Claude）或开源私有化模型（Llama 3、Qwen），但最终交付方案需说明每种模型的成本与隐私风险。
- **初期交付范围**：优先实现文本对话 + 长时记忆 + 角色管理后台（Web 版），移动端小程序作为第二阶段。
### 六、可衡量的成功标准

1. **用户留存**：测试用户连续 7 天与角色对话的比例 ≥ 40%。
2. **角色喜爱度**：用户对角色“像真人”的评分（5 分制）平均 ≥ 4.2 分。
3. **记忆有用性**：用户主动使用“记住”功能的比例 ≥ 15% 的会话。
4. **安全红线**：运营期间 0 次严重内容安全事故。

### 七、交付物要求

- 可演示的 Web 版角色聊天机器人（至少含 2 个预设角色：温柔型 + 幽默型）。
- 角色创建与管理界面（非技术用户可用）。
- 用户对话记录与记忆查看页面。
- 简单 API 文档（供后续游戏引擎或小程序调用）。
- 运维手册（包括模型调用监控、成本统计、内容审核方案）。
